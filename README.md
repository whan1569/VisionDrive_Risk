# MotionRisk Vision: Optical Flow 기반 사고 전조 판단 실험

## 프로젝트 개요

MotionRisk Vision은 카메라 영상과 모바일 센서 데이터를 활용하여 교통 사고 전조 증상을 판단할 수 있는지 검토한 컴퓨터비전 기반 기술 실험 프로젝트이다.

단순 객체 인식이 아니라, 영상 내 움직임 변화와 주변 환경 정보를 분석하여 사고 위험이 증가하는 상황을 사전에 감지하는 것을 목표로 했다.
이를 위해 Optical Flow 기반 움직임 분석, 카메라 영상 처리, React Native Expo 환경에서의 카메라 앱 연결, 모바일 센서 활용 가능성을 검토했다.

## 기술 검토 내용

### 1. Optical Flow 기반 움직임 분석

영상 프레임 간 픽셀 또는 특징점의 이동 흐름을 분석하여 화면 내 객체와 배경의 움직임 변화를 파악하는 방식을 검토했다.

Optical Flow를 활용하면 단순히 객체가 존재하는지만 보는 것이 아니라, 객체가 어느 방향으로 움직이고 있는지, 주변 흐름과 다르게 움직이는 대상이 있는지, 급격한 변화가 발생하는지를 분석할 수 있다.

이를 사고 전조 판단 관점에서 활용하면 다음과 같은 위험 신호를 탐지할 수 있다고 판단했다.

* 급격한 접근 움직임
* 비정상적인 방향 변화
* 주변 흐름과 다른 이동 패턴
* 차선 또는 진행 방향과 다른 움직임
* 충돌 가능성이 높아지는 상대 움직임

---

### 2. Expo / React Native 기반 카메라 앱 검토

모바일 환경에서 실시간 카메라 데이터를 수집하기 위해 Expo 기반 React Native 환경을 검토했다.

검토 결과, React Native 환경에서도 카메라 앱을 구성하고 영상 데이터를 처리하는 구조를 만들 수 있음을 확인했다.
또한 모바일 기기의 센서 데이터를 함께 활용할 수 있어, 카메라 영상만 사용하는 방식보다 더 많은 상황 정보를 얻을 수 있다고 판단했다.

활용 가능성이 확인된 요소는 다음과 같다.

* 카메라 영상 입력
* 실시간 프레임 처리 구조
* 모바일 센서 데이터 활용
* 기기 움직임 정보 활용
* 카메라 영상과 센서 데이터의 결합 가능성

---

### 3. 카메라와 센서 데이터 결합 가능성

사고 전조 판단에서는 영상 정보만으로는 한계가 있을 수 있다.
따라서 카메라 영상과 모바일 센서 데이터를 함께 활용하여 판단 정확도를 높이는 구조를 검토했다.

예를 들어 카메라 영상에서는 전방 객체와 움직임을 분석하고, 센서에서는 기기 흔들림, 방향 변화, 가속도 변화 등을 확인할 수 있다.
이 두 정보를 결합하면 단순 영상 분석보다 더 안정적인 위험도 판단이 가능할 수 있다고 보았다.

```text id="xw0fhn"
카메라 영상 입력
→ Optical Flow 기반 움직임 분석
→ 객체/배경 이동 흐름 파악
→ 모바일 센서 데이터 수집
→ 기기 움직임 및 방향 변화 확인
→ 영상 흐름과 센서 변화 결합
→ 사고 전조 가능성 판단
```

## 프로젝트 판단

이 프로젝트는 기술적으로 불가능해서 중단된 것이 아니다.
오히려 Optical Flow 기반 움직임 분석, Expo/React Native 카메라 앱 구성, 모바일 센서 활용 가능성은 확인했다.

다만 실제 구현 관점에서 다음과 같은 문제가 있었다.

### 1. 직접 구현의 기회비용 문제

사고 전조 판단을 직접 구현하려면 영상 처리, 움직임 분석, 센서 결합, 실시간 처리, 모델 최적화, 모바일 앱 안정화까지 모두 필요했다.

하지만 SAM, SamurAI와 같은 고성능 시각 AI 모델과 기존 비전 기술의 발전 속도를 고려했을 때, 제한된 인력과 시간으로 직접 구현하는 방식은 기회비용이 컸다.

### 2. 효율성 문제

직접 Optical Flow와 센서 데이터를 조합하여 사고 위험 판단 시스템을 만드는 방식은 기술적으로 가능했지만, 현재의 고성능 비전 모델을 활용하는 방식에 비해 효율성이 낮다고 판단했다.

즉, 같은 시간과 비용을 투입했을 때 직접 구현 방식보다 기존 모델이나 상용화된 비전 AI 기술을 활용하는 방식이 더 높은 완성도와 안정성을 낼 가능성이 높았다.

### 3. 하드웨어 구성 비용 문제

정확한 거리감과 상대 움직임을 판단하려면 단일 카메라만으로는 한계가 있었다.
이를 보완하기 위해 2개 이상의 카메라를 사용하는 구조를 검토했지만, 카메라 간 위치와 각도를 안정적으로 고정해야 하고, 장착 구조와 캘리브레이션까지 필요했다.

이 과정에서 하드웨어 비용과 설치 비용이 증가하여 실용성이 떨어진다고 판단했다.

## 프로젝트 결론

MotionRisk Vision은 사고 전조 판단 시스템의 가능성을 검토한 기술 실험 프로젝트이다.

Optical Flow 기반 움직임 분석과 Expo/React Native 기반 카메라 앱 연결, 모바일 센서 활용 가능성은 확인했지만, 현재 기술 수준과 비교했을 때 직접 구현 방식은 기회비용과 효율성 측면에서 불리하다고 판단했다.

따라서 프로젝트는 기술적 실패가 아니라, 구현 가능성을 확인한 뒤 비용 대비 효율이 낮다고 판단하여 보류한 사례이다.
